What

A complete tracking audit in minutes

Odit is a self-hosted auditing tool that runs as a four-container Docker application. It launches a real Chromium browser, routes all traffic through an intercepting proxy, and gives you a full picture of what a website sends — and to whom.

  • ✓ Vendor detection — 40+ analytics, tag managers, pixels, and consent tools identified by domain, script, JS globals, and cookies
  • ✓ Issue detection — 10 built-in rules flag consent violations, PII in URLs, broken scripts, missing CMPs, duplicate signals, and more
  • ✓ GDPR/CCPA posture — run with and without consent interaction to verify what fires before and after the cookie banner
  • ✓ AI-powered analysis — each vendor's network payloads analysed for PII, identifiers, and data flows using Claude
  • ✓ Export everything — Excel workbook, HTML report, Markdown, JSON, HAR files, and per-page screenshots

Where

On your machine.
Nowhere else.

Odit is local-first. It runs as four Docker containers on your laptop or server. Audit data is stored in a local folder. Nothing is uploaded to any cloud service.

🐳 Docker Compose — one command to start everything
💾 Local storage — audit data in ./data/ on your filesystem
🔑 API key stays local — stored in your local database only
🖥️ macOS · Linux · Windows — anywhere Docker Desktop runs
terminal 
git clone https://github.com/venk-hub/Odit.git
cd Odit
# Copy environment file
cp .env.example .env
# Start all containers
./start-auditor.sh
# Open the app
open http://localhost:8000

Requires Docker Desktop. That's it.
