What is Odit?

Odit is a local-first website tracking auditor. It crawls any URL using a real browser, intercepts all network traffic through a proxy, and produces a structured evidence report of every analytics tag, advertising pixel, consent mechanism, and third-party vendor it finds.

Performance Metrics

Each page shows LCP (Largest Contentful Paint — how long until the main content loads), CLS (Cumulative Layout Shift — visual stability), and FID (First Input Delay — interactivity responsiveness). These are the Core Web Vitals used by Google for ranking. Green = good, amber = needs improvement, red = poor.

Setting up your API key: Go to Settings, paste your Anthropic API key, and click Save. The key is stored only in your local database and is never transmitted anywhere except directly to Anthropic's API on your behalf.

No API key? Odit runs fully without AI features. Audits, vendor detection, issue rules, and exports all work without a key. AI is only needed for natural-language Q&A and narrative summaries.

Your API Key — Your Responsibility

If you use the AI assistant features, you supply your own Anthropic API key. Your key is stored locally in your database and is used only to make requests to Anthropic's API directly from your machine. Odit does not store, proxy, or transmit your API key to any third party. You are responsible for your API usage costs, quota limits, and compliance with Anthropic's usage policy. Keep your API key secure — treat it like a password. If compromised, rotate it immediately in your Anthropic console.

What Data Leaves Your Machine

Odit is local-first. The crawler, proxy, database, and web UI all run on your own hardware inside Docker. The only outbound connections are:

  • • Browser traffic to the site you are auditing — as if you visited it normally.
  • • AI chat messages sent to Anthropic's API (only if you have configured a key and use the AI sidebar).
  • • CDN requests for Tailwind CSS, HTMX, and marked.js loaded in your browser from public CDNs.

Audit data, vendor detections, issues, exports, and your API key never leave your machine via Odit.

Responsible Use

Only audit websites you own, are authorised to test, or have explicit permission to crawl. Auditing third-party websites without authorisation may violate their terms of service, computer access laws (e.g. the UK Computer Misuse Act, US CFAA), or other regulations in your jurisdiction. Odit generates real browser traffic to the target site — repeated audits may be noticed by site owners or rate-limiting systems. The authors accept no responsibility for misuse of this tool.
